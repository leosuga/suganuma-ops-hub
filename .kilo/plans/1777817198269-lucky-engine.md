> **Foco**: Corrigir bugs, completar funcionalidades faltantes nos módulos existentes e adicionar mais acionabilidade ao Dashboard.  
> **Plataforma**: Uso equilibrado desktop + mobile — todas as mudanças devem funcionar em ambos.

## Fase 1 — Correção de Bugs (P0)

### 1.1 Streak de protocolo quebrado — `ProtocolList.tsx:60-64`
- **Problema**: o cálculo usa o índice do array (`entries.indexOf(e)`) como offset de data, em vez de verificar dias consecutivos de verdade.
- **Solução**: reescrever a lógica para percorrer os `entries` ordenados por `done_on DESC` e contar dias consecutivos a partir de hoje para trás.
- **Arquivo**: `src/components/health/ProtocolList.tsx`

### 1.2 Mismatch de campo no BiometricLog — `BiometricLog.tsx:12`
- **Problema**: o UI usa `mgdl` como key, mas o schema `glucoseValueSchema` em `health.ts:12` define `mg_dl`. O valor salvo não passa na validação do Zod quando usado via API.
- **Solução**: alinhar para `mg_dl` em ambos os lugares (`BiometricLog.tsx` campos e `formatValue`).
- **Arquivos**: `src/components/health/BiometricLog.tsx`, `src/lib/schemas/health.ts` (verificar consistência)

### 1.3 Agrupamento frágil no RevenueChart — `RevenueChart.tsx:22`
- **Problema**: `txn.occurred_on.slice(8, 10)` assume formato exato `YYYY-MM-DD`. Se a data vier como ISO completa ou outro formato, quebra.
- **Solução**: usar `new Date(txn.occurred_on).getUTCDate()` para extrair o dia de forma robusta.
- **Arquivo**: `src/components/finance/RevenueChart.tsx`

## Fase 2 — Funcionalidades Faltantes (P1)

### 2.1 Edição de Task
- **Problema**: só é possível criar e alternar done/undone. Não há como editar título, categoria, prioridade, data ou notas de uma task existente.
- **Solução**: criar `EditTaskDialog.tsx` reutilizando o padrão do `QuickAddDialog.tsx`:
  - Dialog preenchido com dados atuais da task
  - Campos: título, categoria, prioridade, data, notas (textarea)
  - Botão de deletar task no próprio dialog
  - Abrir via clique no TaskRow (ícone de editar ou duplo clique)
- **Arquivos novos**: `src/components/tasks/EditTaskDialog.tsx`
- **Arquivos modificados**: `src/components/tasks/TaskRow.tsx` (adicionar botão/trigger de edição), `src/app/(app)/tasks/page.tsx` (integrar dialog)
- **Query existente**: `useUpdateTask` já suporta atualizações parciais — ok.

### 2.2 CRUD de Contas (Account)
- **Problema**: `useAccounts` e `useCreateAccount` existem nas queries, mas não há UI para criar/editar/deletar contas. O dropdown de `account_id` no `AddTransactionDialog` e `EditTransactionDialog` não tem fonte de dados gerenciável.
- **Solução**: criar `AccountManager` como componente ou seção na página de Finance:
  - Lista de contas com nome, tipo, moeda, saldo inicial
  - Botão "+ NOVA CONTA" → dialog de criação
  - Botão de editar (nome, tipo) em cada conta
  - Botão de deletar (com confirmação, só se sem transações vinculadas)
  - Opcional: exibir `account_id` nulo no TransactionTable como "(sem conta)"
- **Queries necessárias**: `useUpdateAccount` e `useDeleteAccount` (criar em `finance.ts` seguindo o padrão existente)
- **Arquivos novos**: `src/components/finance/AccountManager.tsx` (ou integrar na página)
- **Arquivos modificados**: `src/lib/queries/finance.ts`, `src/app/(app)/finance/page.tsx`

### 2.3 Edição de Protocolo
- **Problema**: só é possível criar e dar check. Não dá pra renomear, desativar/arquivar ou deletar um protocolo.
- **Solução**: adicionar ações inline em cada `ProtocolRow`:
  - Botão de editar (abre dialog de renomeação)
  - Toggle de ativo/inativo (soft delete via `active = false`)
- **Queries necessárias**: `useUpdateProtocol` (criar em `health.ts`)
- **Arquivos modificados**: `src/lib/queries/health.ts`, `src/components/health/ProtocolList.tsx`

### 2.4 Ações inline no TransactionTable
- **Problema**: os botões de editar/deletar já existem no `TransactionTable` via props `onEdit`/`onDelete`. Funcionam.
- **Melhoria**: adicionar feedback visual durante a deleção (já que `useDeleteTransaction` não tem optimistic update — adicionar para consistência com tasks).
- **Arquivos modificados**: `src/lib/queries/finance.ts` (adicionar optimistic ao `useDeleteTransaction`), `src/components/finance/TransactionTable.tsx` (loading state por linha)

## Fase 3 — Dashboard Acionável (P2)

### 3.1 Quick-Add Task do Dashboard
- **Solução**: adicionar um input inline no topo do Dashboard (estilo "O que você precisa fazer?"), que ao submeter cria a task via `useCreateTask` com categoria `personal` e prioridade `med` como defaults. Um botão "..." ao lado abre o `QuickAddDialog` completo.
- **Arquivo modificado**: `src/app/(app)/dashboard/page.tsx`

### 3.2 Quick-Log Biometria do Dashboard
- **Solução**: adicionar uma mini-seção no Dashboard com os tipos de biometria mais comuns (peso e pressão) como inputs rápidos. Ao submeter, chama `useCreateHealthLog`. Só aparece se houver dados de health configurados (ex: pregnancy ativo).
- **Arquivo modificado**: `src/app/(app)/dashboard/page.tsx`

### 3.3 Quick-Add Transação do Dashboard
- **Solução**: similar ao quick-add task, um input inline para registrar despesa rápida (valor + descrição). Assume `kind=expense`, `occurred_on=hoje`. Um botão "..." abre o `AddTransactionDialog` completo.
- **Arquivo modificado**: `src/app/(app)/dashboard/page.tsx`

## Fase 4 — UX Polish (P3)

### 4.1 Busca textual de tasks
- **Problema**: filtro atual é só por categoria. Para muitas tasks, buscar por texto é essencial.
- **Solução**: adicionar um input de busca no topo da lista de tasks que filtra client-side por `title` e `notes`. Alternativa mais performática: debounced search com query param enviado ao backend.
- **Arquivo modificado**: `src/app/(app)/tasks/page.tsx`

### 4.2 Exibir notas da task
- **Problema**: `TaskRow` só mostra título. Notas ficam invisíveis.
- **Solução**: no `EditTaskDialog` (criado em 2.1), mostrar notas em textarea. Opcional: expandir TaskRow ao clique para revelar notas.
- **Arquivos modificados**: `src/components/tasks/TaskRow.tsx`, `src/components/tasks/EditTaskDialog.tsx`

### 4.3 Consistência de cores e tokens
- **Problema**: `health` usa `#A8D8B0` hardcoded em vários lugares, em vez de um token CSS (`--color-health`). Dificulta manutenção e theming.
- **Solução**: adicionar `--color-health: #A8D8B0` no `globals.css` e referenciar como `text-health`/`border-health`/`bg-health` nos componentes health.
- **Arquivos modificados**: `src/app/globals.css`, todos os componentes de `src/components/health/`

### 4.4 Ordenação por prioridade no Dashboard
- **Problema**: a seção "Tasks por categoria" mostra apenas contagem por categoria. Não mostra as tasks urgentes/atrasadas diretamente.
- **Solução**: adicionar uma subseção "Precisa de atenção" com as 3 tasks mais urgentes/atrasadas, com link para a página de tasks.
- **Arquivo modificado**: `src/app/(app)/dashboard/page.tsx`

### 4.5 Light mode consistente
- **Problema**: `html.light` existe no CSS com valores customizados, mas pode não ter sido testado adequadamente. O `defaultTheme="dark"` no layout força dark no primeiro carregamento, o que é ok, mas ao trocar, certos componentes podem quebrar visualmente.
- **Solução**: testar manualmente o light mode em todas as páginas após as mudanças e ajustar cores onde necessário.
